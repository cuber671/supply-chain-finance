三大业务场景详解                                                                                                                                                               
                                         
  ---                                                                                                                                                                            
  场景一：直接移库 (SCENE_DIRECT_TRANSFER = 1)
                                                                                                                                                                                 
  定义：货主将仓单项下货物从A仓库转移到B仓库，所有权不转移。
                                                                                                                                                                                 
  核心特征：物流过程中仓单锁定 → 交付后解锁，防止在途货物被二次质押。                                                                                                            
   
  完整流程                                                                                                                                                                       
                                                            
  创建委派单 ──→ 锁定仓单（状态:在库→已质押）                                                                                                                                    
      ↓                                                     
  指派司机（状态:待指派→已调度）                                                                                                                                                 
      ↓
  提货确认（状态:已调度→运输中）                                                                                                                                                 
      ↓                                                     
  轨迹上报（可多次，含偏航检测和信用扣分）
      ↓                                                                                                                                                                          
  到货入库（新建仓单 或 合并到已有仓单）
      ↓                                                                                                                                                                          
  确认交付 ──→ 解锁仓单（状态:已质押→在库）                 
                                                                                                                                                                                 
  ┌──────────┬────────────────────────┬────────────────────┬────────────┐                                                                                                        
  │   步骤   │          API           │      仓单操作      │ 委派单状态 │
  ├──────────┼────────────────────────┼────────────────────┼────────────┤                                                                                                        
  │ 创建     │ POST /create           │ lockReceipt 锁定   │ 1-待指派   │
  ├──────────┼────────────────────────┼────────────────────┼────────────┤
  │ 指派     │ POST /assign           │ -                  │ 2-已调度   │                                                                                                        
  ├──────────┼────────────────────────┼────────────────────┼────────────┤
  │ 提货     │ POST /pickup           │ -                  │ 3-运输中   │                                                                                                        
  ├──────────┼────────────────────────┼────────────────────┼────────────┤                                                                                                        
  │ 轨迹上报 │ POST /track/report     │ -                  │ 3-运输中   │
  ├──────────┼────────────────────────┼────────────────────┼────────────┤                                                                                                        
  │ 到货     │ POST /arrive           │ 新建/合并仓单      │ 3-运输中   │
  ├──────────┼────────────────────────┼────────────────────┼────────────┤                                                                                                        
  │ 交付     │ POST /delivery/confirm │ unlockReceipt 解锁 │ 4-已交付   │
  └──────────┴────────────────────────┴────────────────────┴────────────┘                                                                                                        
                                                            
  关键约束

  - receiptId 必填，且属于 ownerEntId                                                                                                                                            
  - sourceWhId 必填，且归属 ownerEntId
  - targetWhId 必填                                                                                                                                                              
  - 失效时若委派单处于非在途状态，必须先解锁仓单（防止仓单永久锁死）                                                                                                             
  - 在途状态（STATUS_IN_TRANSIT）禁止失效                                                                                                                                        
                                                                                                                                                                                 
  ---                                                                                                                                                                            
  场景二：转让后移库 (SCENE_TRANSFER_THEN_TRANSFER = 2)     
                                                       
  定义：仓单背书转让后（货物所有权已转移给受让方），受让方安排物流将货物从原仓库转移到自己控制的仓库。
                                                                                                                                                                                 
  核心特征：不锁定仓单（仓单已在受让方名下），但目标仓库必须属于受让方。
                                                                                                                                                                                 
  完整流程                                                  

  仓单背书转让（仓单模块独立流程）
      ownerHash: 企业A → 企业B                                                                                                                                                   
      ↓
  创建委派单（关联 endorseId，目标仓库归属企业B）                                                                                                                                
      ↓                                                                                                                                                                          
  指派司机
      ↓                                                                                                                                                                          
  提货确认                                                                                                                                                                       
      ↓
  轨迹上报                                                                                                                                                                       
      ↓                                                     
  到货入库
      ↓
  确认交付

  ┌──────────┬────────────────────────────────────────┬────────────────┬────────────┐
  │   步骤   │                  API                   │    仓单操作    │ 委派单状态 │
  ├──────────┼────────────────────────────────────────┼────────────────┼────────────┤                                                                                            
  │ 背书转让 │ launchEndorsement + confirmEndorsement │ ownerHash 变更 │ -          │
  ├──────────┼────────────────────────────────────────┼────────────────┼────────────┤                                                                                            
  │ 创建     │ POST /create                           │ 不锁定         │ 1-待指派   │
  ├──────────┼────────────────────────────────────────┼────────────────┼────────────┤                                                                                            
  │ 指派     │ POST /assign                           │ -              │ 2-已调度   │
  ├──────────┼────────────────────────────────────────┼────────────────┼────────────┤                                                                                            
  │ 提货     │ POST /pickup                           │ -              │ 3-运输中   │
  ├──────────┼────────────────────────────────────────┼────────────────┼────────────┤                                                                                            
  │ 轨迹上报 │ POST /track/report                     │ -              │ 3-运输中   │
  ├──────────┼────────────────────────────────────────┼────────────────┼────────────┤                                                                                            
  │ 到货     │ POST /arrive                           │ 新建/合并仓单  │ 3-运输中   │
  ├──────────┼────────────────────────────────────────┼────────────────┼────────────┤                                                                                            
  │ 交付     │ POST /delivery/confirm                 │ 不操作仓单     │ 4-已交付   │
  └──────────┴────────────────────────────────────────┴────────────────┴────────────┘                                                                                            
                                                            
  关键约束

  - endorseId 必填（关联背书记录）                                                                                                                                               
  - targetWhId 必填，且必须属于受让方（ownerEntId）
  - 失效时不涉及仓单操作（仓单未锁定）                                                                                                                                           
  - 交付时仓单服务无操作（背书已完成所有权变更）                                                                                                                                 
                                                                                                                                                                                 
  ---                                                                                                                                                                            
  场景三：发货入库 (SCENE_DELIVERY_TO_WAREHOUSE = 3)        
                                                                                                                                                                                 
  定义：货物从供应商发往仓库，到货后创建新仓单（如粮食从农户收购点运往国家粮库）。
                                                                                                                                                                                 
  核心特征：无仓单关联，到货时创建新仓单；无锁定/解锁操作。                                                                                                                      
                                                                                                                                                                                 
  完整流程                                                                                                                                                                       
                                                            
  创建委派单（无 receiptId，仅指定 targetWhId）
      ↓                                                                                                                                                                          
  指派司机
      ↓                                                                                                                                                                          
  提货确认                                                  
      ↓
  轨迹上报
      ↓
  到货入库 ──→ 创建新仓单（mintDirectReceipt）
      ↓                                                                                                                                                                          
  确认交付
                                                                                                                                                                                 
  ┌──────────┬────────────────────────┬──────────────────────────────┬────────────┐
  │   步骤   │          API           │           仓单操作           │ 委派单状态 │
  ├──────────┼────────────────────────┼──────────────────────────────┼────────────┤                                                                                              
  │ 创建     │ POST /create           │ 无                           │ 1-待指派   │
  ├──────────┼────────────────────────┼──────────────────────────────┼────────────┤                                                                                              
  │ 指派     │ POST /assign           │ -                            │ 2-已调度   │
  ├──────────┼────────────────────────┼──────────────────────────────┼────────────┤
  │ 提货     │ POST /pickup           │ -                            │ 3-运输中   │
  ├──────────┼────────────────────────┼──────────────────────────────┼────────────┤                                                                                              
  │ 轨迹上报 │ POST /track/report     │ -                            │ 3-运输中   │
  ├──────────┼────────────────────────┼──────────────────────────────┼────────────┤                                                                                              
  │ 到货     │ POST /arrive           │ mintDirectReceipt 创建新仓单 │ 3-运输中   │
  ├──────────┼────────────────────────┼──────────────────────────────┼────────────┤                                                                                              
  │ 交付     │ POST /delivery/confirm │ 无                           │ 4-已交付   │
  └──────────┴────────────────────────┴──────────────────────────────┴────────────┘                                                                                              
                                                            
  到货入库的两种方式

  ┌───────────────────────────────────┬──────────────────────────┬───────────────────┐                                                                                           
  │            actionType             │           说明           │       调用        │
  ├───────────────────────────────────┼──────────────────────────┼───────────────────┤                                                                                           
  │ ACTION_CREATE_NEW_RECEIPT = 1     │ 全量入库，创建新仓单     │ mintDirectReceipt │
  ├───────────────────────────────────┼──────────────────────────┼───────────────────┤
  │ ACTION_MERGE_EXISTING_RECEIPT = 2 │ 增量入库，合并到已有仓单 │ mergeReceipt      │
  └───────────────────────────────────┴──────────────────────────┴───────────────────┘                                                                                           
   
  关键约束                                                                                                                                                                       
                                                            
  - receiptId 不填（无原有仓单）                                                                                                                                                 
  - sourceWhId 可为空（默认为0）
  - targetWhId 必填（目的地仓库）                                                                                                                                                
  - 失效时无仓单操作

场景1直接移库 — 完整业务逻辑（实施版）
核心设计原则
原则	说明
action=1 仓储方主导	arrive创建入库申请，仓储方确认+签发仓单，delivery仅作废原仓单
action=2 splitReceipt统一	arrive创建已确认入库单，splitReceipt统一创建新仓单
审计链条完整	入库申请作为审计凭证存在
避免重复仓单	仓单创建路径唯一
接口参数设计
arrive 接口

POST /api/v1/logistics/arrive
{
  "voucherNo": "xxx",           // 委派单编号
  "actionType": 1,              // 1=全量交付，2=部分交付
  "warehouseId": 5,             // 到货仓库ID
  "arrivedWeight": 100          // 到货重量
}
delivery/confirm 接口

POST /api/v1/logistics/delivery/confirm
{
  "voucherNo": "xxx",
  "action": 1                   // 1=全量交付，2=部分交付
}
action=1 全量交付

Step 1: 货主创建委派单
  POST /api/v1/logistics/create
  {
    "businessScene": 1,
    "receiptId": 原仓单ID(100吨, sourceWh=3),
    "sourceWhId": 3,
    "targetWhId": 5,
    "transportQuantity": 100,
    "unit": "吨",
    "carrierEntId": 物流企业ID
  }
  → 仓单状态: IN_STOCK → WAIT_LOGISTICS
  → 委派单状态: PENDING(1)

Step 2: 物流公司指派司机
  POST /api/v1/logistics/assign
  { "voucherNo": "xxx", "driverId": "D001", "driverName": "张三", "vehicleNo": "粤B12345" }
  → 委派单状态: ASSIGNED(2)

Step 3: 司机提货确认
  POST /api/v1/logistics/pickup
  { "voucherNo": "xxx", "authCode": "THX9A6EK" }
  → 委派单状态: ASSIGNED(2) → IN_TRANSIT(3)

Step 4: 物流公司到货登记
  POST /api/v1/logistics/arrive
  {
    "voucherNo": "xxx",
    "actionType": 1,
    "warehouseId": 5,
    "arrivedWeight": 100
  }
  → 创建 targetWh 入库申请（status=1 待确认）
  → 委派单状态: IN_TRANSIT(3)

Step 5: 仓储方确认入库 + 签发仓单
  POST /api/v1/warehouse/stock-in/{stockOrderId}/confirm?warehouseId=5
  → confirmStockOrder 内部:
     1. 更新入库单 status=2 已确认
     2. 自动调用 mintReceipt 签发仓单
  → 新仓单A: 100吨 IN_STOCK (targetWh=5)

Step 6: 货主确认交付
  POST /api/v1/logistics/delivery/confirm
  {
    "voucherNo": "xxx",
    "action": 1
  }
  → 清除原仓单 WAIT_LOGISTICS
  → 作废原仓单
  → 委派单状态: DELIVERED(4)

结果：
  - 新仓单A: 100吨 IN_STOCK (targetWh=5) ✅  仓储方签发
  - 原仓单: 已作废 ✅
action=2 部分交付

Step 1: 货主创建委派单
  POST /api/v1/logistics/create
  {
    "businessScene": 1,
    "receiptId": 原仓单ID(100吨, sourceWh=3),
    "sourceWhId": 3,
    "targetWhId": 5,
    "transportQuantity": 30,
    "unit": "吨",
    "carrierEntId": 物流企业ID
  }
  → 仓单状态: IN_STOCK → WAIT_LOGISTICS
  → 委派单状态: PENDING(1)

Step 2: 物流公司指派司机
  POST /api/v1/logistics/assign
  → 委派单状态: ASSIGNED(2)

Step 3: 司机提货确认
  POST /api/v1/logistics/pickup
  → 委派单状态: ASSIGNED(2) → IN_TRANSIT(3)

Step 4: 物流公司到货登记（目标仓库）
  POST /api/v1/logistics/arrive
  {
    "voucherNo": "xxx",
    "actionType": 2,
    "warehouseId": 5,
    "arrivedWeight": 30
  }
  → 创建 targetWh 入库申请（status=2 已确认，跳过仓储方操作）
  → 不触发自动签发仓单
  → 委派单状态: IN_TRANSIT(3)

Step 5: 物流公司到货登记（源仓库）
  POST /api/v1/logistics/arrive
  {
    "voucherNo": "xxx",
    "actionType": 2,
    "warehouseId": 3,
    "arrivedWeight": 70
  }
  → 创建 sourceWh 入库申请（status=2 已确认）
  → 不触发自动签发仓单
  → 委派单状态: IN_TRANSIT(3)

Step 6: 货主确认交付（触发仓单拆分）
  POST /api/v1/logistics/delivery/confirm
  {
    "voucherNo": "xxx",
    "action": 2
  }
  → 调用 splitReceipt(原仓单ID, weights=[30, 70], warehouseIds=[5, 3])
  → 原仓单拆分:
     - 新仓单B: 30吨 IN_STOCK (warehouseId=5, targetWh)
     - 新仓单C: 70吨 IN_STOCK (warehouseId=3, sourceWh)
  → 原仓单: 已作废
  → 清除 WAIT_LOGISTICS
  → 委派单状态: DELIVERED(4)

结果：
  - 入库申请1: 30吨 status=2 已确认（targetWh）✅
  - 入库申请2: 70吨 status=2 已确认（sourceWh）✅
  - 新仓单B: 30吨 IN_STOCK (targetWh=5) ✅ splitReceipt创建
  - 新仓单C: 70吨 IN_STOCK (sourceWh=3) ✅ splitReceipt创建
  - 原仓单: 已作废 ✅
