金融模块（Finance Service）三大业务场景详解
================================================================================

---

场景一：仓单质押贷款 (SCENE_PLEDGE_LOAN)

定义：企业以持有的大宗商品仓单作为抵押物，向金融机构申请贷款融资。

核心特征：仓单在贷款周期内被锁定（无法流转/背书），还款结清后自动解锁。
           质押率70%，年利率8%，逾期日罚息0.05%。

完整流程

  贷款申请 ──→ 金融机构审批 ──→ 审批通过 ──→ 放款（锁定仓单）
      ↓                                                        ↓
  申请拒绝                                               仓单LOCKED
      ↓                                                        ↓
  [已拒绝]                                            借款方使用资金
                                                          ↓
  按时还款 ─────────────────────────→ 部分还款 ──→ 逾期标记
      ↓                                        ↓           ↓
  解锁仓单                                    [还款中]   [逾期]
  [已结清]                                     ↓           ↓
                                          继续还款   宽限期7天后
                                              ↓        [违约]
                                          [已结清]     ↓
                                                    [违约处置]

│   步骤   │              API               │       仓单操作        │  贷款状态 │
├──────────┼────────────────────────────────┼──────────────────────┼────────────┤
│ 申请     │ POST /loan/apply               │ -                    │ 1-待审批   │
├──────────┼────────────────────────────────┼──────────────────────┼────────────┤
│ 审批通过 │ POST /loan/{id}/approve        │ -                    │ 4-待放款   │
├──────────┼────────────────────────────────┼──────────────────────┼────────────┤

├──────────┼────────────────────────────────┼──────────────────────┼────────────┤
│ 取消     │ POST /loan/{id}/cancel         │ -                    │ 3-已取消   │
├──────────┼────────────────────────────────┼──────────────────────┼────────────┤

├──────────┼────────────────────────────────┼──────────────────────┼────────────┤

│ 逾期处理 │ LoanOverdueCheckTask (每日1AM) │ -                    │ 8-逾期     │
└──────────┴────────────────────────────────┴──────────────────────┴────────────┘

关键约束

- 申请金额 ≤ 仓单评估价值 × 70%（质押率）
- 仓单状态必须为「已确认」（STATUS_CONFIRMED）
- 放款前校验借款方信用额度（credit-service）
- 放款时锁定仓单（仓单在贷款期间无法背书转让）
- 还款结清后必须解锁仓单（否则仓单永久锁死）
- 支持五种还款方式：CASH / RECEIPT_OFFSET / RECEIVABLE_OFFSET / COLLATERAL_DISPOSAL / PARTIAL

与其他模块的关系

  logistics-service ──→ 提供物流托单信息（货物数量、债务方）
         ↓
  warehouse-service ──→ 仓单查询 + 锁定/解锁（放款锁，还清解）
         ↓
  credit-service ─────→ 审批前校验借款方可用信用额度
         ↓
  enterprise-service ─→ 验证当前用户是否为金融机构
         ↓
  fisco-gateway-service ──→ 区块链合约调用（LoanCore.sol / LoanRepayment.sol）

场景二：应收账款保理融资 (SCENE_RECEIVABLE_FACTORING)

定义：企业将未到期的应收账款转让给保理公司（金融机构），提前获得融资。

核心特征：应收账款一旦融资（isFinanced=1），债权转移给保理公司，原债权人不可再做转让。

完整流程

  物流托单生成 ──→ 应收款创建 ──→ 债务方确认 ──→ 融资（保理）
      ↓              ↓               ↓               ↓
   卖方发货      [待确认]       [已生效]        [已融资]
                                                     ↓
                                          保理公司拥有债权
                                                     ↓
                                          到期日 ──→ 融资方收款 ──→ [已结清]

│   步骤   │              API               │       仓单操作        │   应收状态   │
├──────────┼────────────────────────────────┼──────────────────────┼──────────────┤
│ 创建     │ POST /receivable/generate      │ -                    │ 1-待确认     │
├──────────┼────────────────────────────────┼──────────────────────┼──────────────┤
│ 确认     │ POST /receivable/confirm       │ -                    │ 2-已生效     │
├──────────┼────────────────────────────────┼──────────────────────┼──────────────┤
│ 调整     │ PATCH /receivable/adjust       │ -                    │ 不变         │
├──────────┼────────────────────────────────┼──────────────────────┼──────────────┤
│ 现金还款 │ POST /receivable/repayment/cash│ -                    │ 3-部分还款   │

├──────────┼────────────────────────────────┼──────────────────────┼──────────────┤

│          │ offset                         │                      │              │
├──────────┼────────────────────────────────┼──────────────────────┼──────────────┤
│ 融资     │ POST /receivable/finance       │ -                    │ 标记isFinanced│
├──────────┼────────────────────────────────┼──────────────────────┼──────────────┤
│ 结清     │ POST /receivable/{id}/settle   │ -                    │ 4-已结清     │
└──────────┴────────────────────────────────┴──────────────────────┴──────────────┘

关键约束

- 应收账款来源：物流托单（SCENE_STOCK_IN=入库 / SCENE_TRANSFER_ACCEPT=转让交货）
- 债务方必须签字确认（signature）后才生效
- 融资后 isFinanced=1，债权转移给保理公司
- 融资金额 ≤ 应收账款余额
- 还款可使用仓单抵债（债务方用自己持有的仓单抵消）

与其他模块的关系

  logistics-service ──→ 获取托单信息触发应收款生成
         ↓
  warehouse-service ──→ 仓单抵债时校验仓单归属
         ↓
  fisco-gateway-service ──→ 区块链合约调用（ReceivableCore.sol / ReceivableRepayment.sol）

场景三：仓单抵债还款 (SCENE_RECEIPT_OFFSET)

定义：债务方使用自己持有的仓单，抵消所欠的应收账款或贷款。

核心特征：仓单价值作为还款凭证，不需要现金流转；仓单从债务方过户到债权人。

完整流程（以应收账款抵债为例）

  债务方发起抵债 ──→ 校验仓单归属 ──→ 创建抵债记录 ──→ 更新应收款余额
      ↓                  ↓                  ↓                ↓
   仓单过户           仓单属于债务方     上链记录          部分/全额结清
   (receipt offset)                      (区块链)          (区块链)

┌──────────┬────────────────────────────────┬──────────────────────┬──────────────┐
│   场景   │              API               │       仓单操作        │    状态      │
├──────────┼────────────────────────────────┼──────────────────────┼──────────────┤
│ 贷款还债 │ POST /loan/{id}/repay          │ -                    │ 贷款状态变更 │
│          │ (type=RECEIPT_OFFSET)          │                      │              │
├──────────┼────────────────────────────────┼──────────────────────┼──────────────┤
│ 应收款   │ POST /receivable/repayment/    │ 校验仓单属于债务方    │ 应收款       │
│ 抵债     │ offset                         │                      │ 余额减少     │
└──────────┴────────────────────────────────┴──────────────────────┴──────────────┘

关键约束

- 仓单必须属于债务方（ownerEntId = debtorEntId）
- 抵债价格由双方协商（offsetPrice）
- 抵债金额不能超过剩余欠款
- 抵债记录上链（LoanRepayment.sol / ReceivableRepayment.sol）

与其他模块的关系

  warehouse-service ──→ 查询仓单信息，校验仓单归属
         ↓
  fisco-gateway-service ──→ 区块链合约调用（LoanRepayment.sol / ReceivableRepayment.sol）

关键业务规则汇总
================================================================================

【贷款业务规则】
1. 质押率 70%（DEFAULT_PLEDGE_RATE）
2. 默认年利率 8%（DEFAULT_INTEREST_RATE）
3. 日罚息率 0.05%（DAILY_PENALTY_RATE）
4. 宽限期 7 天（GRACE_PERIOD_DAYS）
5. 放款时锁定仓单，结清时解锁仓单
6. 审批前必须校验借款方可用信用额度

【应收账款业务规则】
1. 应收账款来源：物流托单（入库/转让场景）
2. 必须债务方签字确认后才生效
3. 融资后（isFinanced=1）不可再做转让
4. 支持仓单抵债还款

【仓单角色】
1. 贷款抵押物：放款锁定，结清解锁
2. 抵债工具：债务方用仓单抵消欠款
3. 仓单在锁定期间无法背书转让

【风险控制】
1. 贷款审批前校验借款方信用额度
2. 放款前确认仓单有效且未被其他贷款占用
3. 逾期超过宽限期（7天）标记违约
4. 逾期由定时任务（每日1AM）自动检查并上链
